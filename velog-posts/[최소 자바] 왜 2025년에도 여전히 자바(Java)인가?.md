<blockquote>
<p><strong>&quot;백엔드 개발자가 알아야 하는 최소한의 자바&quot;</strong> 시리즈의 첫 번째 글입니다.</p>
<p>문법은 뗐지만 실무 코드가 두려운 주니어, 다른 언어에서 넘어온 이직자를 위해 작성되었습니다.</p>
</blockquote>

<h3 id="요즘-누가-자바-쓰나요">&quot;요즘 누가 자바 쓰나요?&quot;</h3>
<p>&quot;파이썬(Python)이나 고(Go), 아니면 코틀린(Kotlin)이 대세 아닌가요?&quot;</p>
<p>개발자 커뮤니티나 멘토링을 하다 보면 심심치 않게 듣는 질문입니다. 2024년을 지나 2025년을 바라보는 지금, AI 붐으로 파이썬이 날아오르고 프론트엔드 생태계가 급변하는 와중에 자바는 어쩐지 <strong>'오래된 양복을 입은 아저씨'</strong> 같은 느낌을 주기도 합니다.</p>
<p>하지만 결론부터 말씀드리겠습니다.</p>
<p><strong>백엔드 개발자로 커리어를 시작하거나 성장하고 싶다면, 자바는 여전히 가장 강력하고 안전한 선택지입니다.</strong></p>
<p>단순히 &quot;한국은 자바 공화국이라서(SI가 많아서)&quot;라는 이유 때문만은 아닙니다. 오늘은 우리가 이 언어를 왜 다시 '제대로' 들여다봐야 하는지, 그리고 **Java 8을 넘어 Modern Java(17, 21)**가 어떻게 진화했는지 이야기해 보려 합니다.</p>
<br />

<h2 id="1-지루해서-위대한-언어-boring-is-good">1. 지루해서 위대한 언어 (Boring is Good)</h2>
<p>스타트업 초기 단계에서는 개발 속도가 빠른 Node.js나 Django(Python)가 매력적일 수 있습니다. 하지만 서비스의 규모가 커지고 트래픽이 폭발하며, 수십 명의 개발자가 협업해야 하는 '엔터프라이즈' 단계에 들어서면 이야기가 달라집니다.</p>
<p>이때부터는 <strong>'창의성'보다 '안정성'과 '유지보수성'이 훨씬 중요해집니다.</strong></p>
<h3 id="✅-강력한-정적-타입과-하위-호환성">✅ 강력한 정적 타입과 하위 호환성</h3>
<p>자바는 아주 보수적인 언어입니다. 10년 전에 작성된 코드가 최신 JVM 위에서도 (거의) 문제없이 돌아갑니다. 이는 기업 입장에서 엄청난 장점입니다. 또한, 컴파일 타임에 오류를 잡아주는 강력한 타입 시스템은 대규모 리팩토링 시 개발자의 실수를 막아주는 든든한 보험이 됩니다.</p>
<h3 id="✅-압도적인-생태계-spring-boot">✅ 압도적인 생태계 (Spring Boot)</h3>
<p>&quot;자바가 좋아서 쓴다기보다 스프링이 좋아서 자바를 쓴다&quot;는 말이 있을 정도입니다. Spring Framework와 Spring Boot는 수십 년간 전 세계 개발자들이 쌓아올린 <strong>'검증된 아키텍처'의 집약체</strong>입니다. 보안, 데이터베이스 접근, 모니터링 등 서버 개발에 필요한 모든 것이 이미 준비되어 있습니다.</p>
<br />

<h2 id="2-우리가-써야-할-자바-8-11-17-그리고-21">2. 우리가 써야 할 자바: 8, 11, 17, 그리고 21</h2>
<p>&quot;저는 자바 8 쓰는데요?&quot;라고 하신다면, 이제는 정말 놓아주어야 할 때입니다.</p>
<p>자바는 6개월마다 새 버전이 나오지만, 기업 실무에서는 장기 지원 버전인 <strong>LTS(Long Term Support)</strong> 버전을 기준으로 움직입니다. 주요 LTS 버전의 핵심 변화를 요약해 드립니다.</p>
<h3 id="☕️-java-8-2014---혁신의-시작">☕️ Java 8 (2014) - &quot;혁신의 시작&quot;</h3>
<ul>
<li><strong>Lambda &amp; Stream:</strong> 함수형 프로그래밍 스타일 도입. 코드가 간결해지고 데이터 처리가 쉬워졌습니다.</li>
<li><strong>Optional:</strong> 지긋지긋한 <code>NullPointerException</code>과의 전쟁을 선포했습니다.</li>
<li><em>현황:</em> 여전히 많은 레거시 시스템이 8에 머물러 있지만, 점차 걷어내는 추세입니다.</li>
</ul>
<h3 id="☕️-java-11-2018---안정화-단계">☕️ Java 11 (2018) - &quot;안정화 단계&quot;</h3>
<ul>
<li><strong>String 메서드 강화:</strong> <code>isBlank()</code>, <code>strip()</code>, <code>lines()</code> 등 문자열 처리가 편해졌습니다.</li>
<li><strong>HTTP Client:</strong> 구닥다리 <code>HttpURLConnection</code> 대신 모던한 클라이언트가 표준으로 들어왔습니다.</li>
</ul>
<h3 id="☕️-java-17-2021---새로운-표준-the-new-standard-✨">☕️ Java 17 (2021) - &quot;새로운 표준 (The New Standard)&quot; ✨</h3>
<p>지금 스프링 부트 3.0 이상(Spring Boot 3.x)을 쓰려면 <strong>Java 17 이상이 필수</strong>입니다.</p>
<ul>
<li><strong>Record:</strong> DTO를 만들 때 <code>get</code>, <code>set</code> 없이 데이터 클래스를 한 줄로 정의할 수 있습니다.</li>
<li><strong>Text Blocks:</strong> SQL이나 JSON 문자열을 <code>&quot;&quot;&quot;</code> 안에 줄바꿈 그대로 넣을 수 있습니다.</li>
<li><strong>Switch Expression:</strong> 값을 반환하는 깔끔한 switch 문을 쓸 수 있습니다.</li>
<li><em>추천:</em> <strong>신규 프로젝트를 시작한다면 최소 17 버전을 선택하세요.</strong></li>
</ul>
<h3 id="☕️-java-21-2023---게임-체인저-🚀">☕️ Java 21 (2023) - &quot;게임 체인저&quot; 🚀</h3>
<ul>
<li><strong>Virtual Threads (가상 스레드):</strong> 기존 스레드 모델의 한계를 부수고, 적은 리소스로 수백만 개의 요청을 동시에 처리할 수 있는 기능이 정식 도입되었습니다.</li>
<li><em>현황:</em> 얼리어답터 기업들이 빠르게 도입 중이며, 자바의 성능을 한 단계 끌어올린 버전으로 평가받습니다.</li>
</ul>

<h2 id="3-이-시리즈의-목표">3. 이 시리즈의 목표</h2>
<p>서론이 길었습니다. 이 시리즈는 단순히 문법을 나열하는 강의가 아닙니다.
<strong>&quot;스프링 부트로 서버를 띄울 수는 있지만, 내부가 어떻게 돌아가는지 찜찜한&quot;</strong> 분들을 위해 기획했습니다.</p>
<p>앞으로 우리는 8주에 걸쳐 다음 내용들을 파헤칠 것입니다.</p>
<ol>
<li><strong>JVM:</strong> 내 코드가 메모리에 올라갈 때 벌어지는 일 (면접용 암기 X)</li>
<li><strong>OOP:</strong> 스프링 빈은 왜 싱글톤이고, 인터페이스는 왜 쓸까?</li>
<li><strong>Modern Java:</strong> 람다, 스트림, 옵셔널 '제대로' 쓰기</li>
<li><strong>Collections:</strong> 무지성 ArrayList 멈춰!</li>
<li><strong>Exception:</strong> 에러를 우아하게 처리하는 법</li>
<li><strong>Concurrency:</strong> 멀티 스레드와 동시성 이슈 정복</li>
<li><strong>Java 14+:</strong> 롬복을 걷어낼 수 있는 신문법들</li>
<li><strong>Generics &amp; Annotation:</strong> 라이브러리 코드를 읽는 힘 기르기</li>
</ol>
