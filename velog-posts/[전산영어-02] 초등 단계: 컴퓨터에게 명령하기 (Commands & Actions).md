<p><strong>[Step 2. 초등 단계]</strong> 본문입니다.</p>
<p>초등학생이 되면 &quot;밥 먹어&quot;, &quot;학교 가&quot;처럼 동사를 사용해 문장을 만들기 시작하죠. 컴퓨터도 똑같습니다. 명칭(Step 1)을 알았으니, 이제 컴퓨터에게 <strong>'행동(Action)'</strong>을 시키는 동사들을 배울 차례입니다.</p>
<p>개발자들이 입버릇처럼 쓰는 <code>Run</code>, <code>Load</code> 같은 단어들의 정확한 뉘앙스를 담았습니다.</p>
<hr />
<p>지난 시간(유치원 단계)에는 컴퓨터 부품과 화면의 <strong>'이름(명사)'</strong>을 배웠습니다.
하지만 이름만 안다고 컴퓨터를 부려먹을 수는 없죠.</p>
<blockquote>
<p>&quot;야, 컴퓨터!&quot; (묵묵부답)
&quot;야, 컴퓨터! <strong>실행해!</strong>&quot; (윙~)</p>
</blockquote>
<p>이번 <strong>초등 단계</strong>에서는 컴퓨터를 움직이게 만드는 <strong>'동사(Verb)'</strong>들을 배웁니다. 우리가 흔히 쓰는 단어지만, 컴퓨터 세계에서는 조금 더 깊은 뜻을 가지고 있습니다.</p>
<br />

<h2 id="1-기억하고-불러오기-memory">1. 기억하고 불러오기 (Memory)</h2>
<p>가장 기본은 무언가를 만들고, 보관하고, 꺼내는 일입니다.</p>
<h3 id="💾-save-세이브">💾 Save (세이브)</h3>
<ul>
<li><strong>사전적 의미:</strong> (위험에서) 구하다, 저축하다</li>
<li><strong>전산적 의미:</strong> <strong>저장하다</strong></li>
<li><strong>설명:</strong> 왜 'Save(구하다)'일까요? 컴퓨터가 꺼지면 작업하던 내용은 다 날아갑니다(죽습니다). 그 전에 하드디스크라는 안전한 곳으로 옮겨서 <strong>'목숨을 구해주는 것'</strong>이 바로 저장(Save)입니다.</li>
</ul>
<h3 id="🚛-load-로드">🚛 Load (로드)</h3>
<ul>
<li><strong>사전적 의미:</strong> (짐을) 싣다</li>
<li><strong>전산적 의미:</strong> <strong>불러오기</strong></li>
<li><strong>설명:</strong> 우리는 흔히 파일을 '연다(Open)'고 하죠? 개발자들은 <strong>'로딩(Loading)'</strong>한다고 합니다.<ul>
<li>창고(하드디스크)에 있는 데이터를 꺼내서, 작업대(메모리)라는 트럭에 <strong>'짐을 싣는(Load)'</strong> 과정이기 때문입니다.</li>
<li>*&quot;게임 로딩 중...&quot;* = *&quot;게임 데이터를 메모리에 싣는 중...&quot;*</li>
</ul>
</li>
</ul>
<h3 id="📋-paste-페이스트">📋 Paste (페이스트)</h3>
<ul>
<li><strong>사전적 의미:</strong> 풀, 반죽 / 풀로 붙이다</li>
<li><strong>설명:</strong> '복사(Copy)'는 다들 아시죠? 붙여넣기가 영어로 <strong>Paste</strong>인 이유는, 종이에 풀칠해서 턱! 붙이는 모습을 상상하면 됩니다.</li>
</ul>
<br />

<h2 id="2-프로그램-움직이기-control">2. 프로그램 움직이기 (Control)</h2>
<p>이제 프로그램을 쥐락펴락해봅시다.</p>
<h3 id="🏃-run-런">🏃 Run (런)</h3>
<ul>
<li><strong>사전적 의미:</strong> 달리다</li>
<li><strong>전산적 의미:</strong> <strong>(프로그램을) 실행하다</strong></li>
<li><strong>설명:</strong> 개발자들이 가장 많이 쓰는 단어 중 하나입니다. 코드를 다 짰으면 컴퓨터에게 *&quot;자, 이제 달려봐(작동해봐)!&quot;* 라고 명령하는데, 이걸 <strong>&quot;코드를 런(Run)한다&quot;</strong>라고 표현합니다.</li>
</ul>
<h3 id="📦-install-인스톨-vs-uninstall-언인스톨">📦 Install (인스톨) vs Uninstall (언인스톨)</h3>
<ul>
<li><strong>Install:</strong> (장비 등을) 설치하다. 자리를 잡게 하다.</li>
<li><strong>Uninstall:</strong> <code>Un(반대)</code> + <code>Install</code>. 설치된 것을 제거하다.</li>
<li><strong>설명:</strong> 프로그램 깔 때 <code>Setup</code>이라고도 하지만, <code>Install</code>이 더 전문적인 표현입니다. 지울 때는 <code>Delete</code>보다 <strong><code>Uninstall</code></strong>을 씁니다. 단순 삭제가 아니라, 설치했던 모든 흔적을 되돌린다는 뜻이 강합니다.</li>
</ul>
<h3 id="🚪-exit--quit--close-종료-3형제">🚪 Exit / Quit / Close (종료 3형제)</h3>
<ul>
<li><strong>Close (닫다):</strong> 현재 보고 있는 창(Window)만 닫을 때 주로 씁니다. (프로그램은 뒤에서 계속 돌고 있을 수도 있음)</li>
<li><strong>Exit (출구/나가다):</strong> 프로그램 자체를 완전히 끝내고 나갈 때 씁니다.</li>
<li><strong>Quit (그만두다):</strong> Exit와 비슷하지만, &quot;나 이제 일 안 해!&quot; 하고 손 떼는 느낌입니다. (맥OS에서 주로 사용)</li>
</ul>
<br />

<h2 id="3-컴퓨터의-대답-듣기-status">3. 컴퓨터의 대답 듣기 (Status)</h2>
<p>명령을 내리면 컴퓨터는 현재 상태를 보고합니다.</p>
<h3 id="⏳-idle-아이들">⏳ Idle (아이들)</h3>
<ul>
<li><strong>의미:</strong> 게으른, 나태한, 가동되지 않는</li>
<li><strong>설명:</strong> 컴퓨터나 프로그램이 아무 일도 안 하고 멍하니 대기하고 있는 상태를 <strong>Idle</strong>이라고 합니다.<ul>
<li>*&quot;왜 CPU가 놀고 있지?&quot;* = *&quot;CPU 상태가 <strong>Idle</strong>이네?&quot;*</li>
</ul>
</li>
</ul>
<h3 id="✅-success-석세스-vs-❌-fail-페일">✅ Success (석세스) vs ❌ Fail (페일)</h3>
<ul>
<li><strong>Success:</strong> 성공. (명령을 잘 수행함)</li>
<li><strong>Fail:</strong> 실패. (명령 수행 못 함)</li>
<li><strong>Failure:</strong> 고장, 실패.</li>
</ul>
<h3 id="⚠️-error-에러">⚠️ Error (에러)</h3>
<ul>
<li><strong>의미:</strong> 실수, 오류</li>
<li><strong>설명:</strong> 컴퓨터가 &quot;주인님, 시키신 일을 못 하겠어요!&quot;라고 외치는 비명입니다.<ul>
<li><strong>Fatal Error:</strong> 치명적인(Fatal) 오류. 프로그램이 죽어버릴 정도로 심각한 상태입니다.</li>
</ul>
</li>
</ul>
<br />

<h2 id="🍎-요약-summary">🍎 요약 (Summary)</h2>
<p>오늘은 컴퓨터를 부리는 주문(동사)을 배웠습니다.</p>
<ol>
<li>파일을 여는 것은 짐을 싣는 것(<strong>Load</strong>)과 같다.</li>
<li>코드를 작동시키는 것은 달리게 하는 것(<strong>Run</strong>)이다.</li>
<li>컴퓨터가 놀고 있는 상태는 <strong>Idle</strong>이다.</li>
</ol>
<p>다음 단계부터는 진짜 코딩의 세계로 들어갑니다.
<strong>[Step 3. 중등 단계: 데이터의 종류]</strong>에서는 <code>String</code>, <code>Integer</code>, <code>Boolean</code> 같은, 프로그래밍 언어 책 첫 장에 나오는 알쏭달쏭한 단어들을 아주 쉽게 파헤쳐 봅니다.</p>
