<p>유치원생이 세상을 배울 때, 가장 먼저 &quot;이건 사과&quot;, &quot;저건 자동차&quot;라고 사물의 <strong>이름</strong>을 익히는 것처럼, 컴퓨터의 가장 기초적인 부품과 화면 구성요소의 이름부터 확실하게 잡고 가는 글입니다.</p>
<hr />
<p>전산 영어 완전 정복, 그 첫 번째 시간은 <strong>'유치원 단계'</strong>입니다.</p>
<p>우리가 영어를 처음 배울 때를 떠올려보세요. 문법부터 배우지 않습니다. 눈에 보이는 사물을 가리키며 &quot;Apple(사과)&quot;, &quot;Desk(책상)&quot; 같은 <strong>이름(명사)</strong>부터 익힙니다.</p>
<p>컴퓨터도 똑같습니다. 복잡한 코드 이전에, 당장 내 눈앞에 있는 기계와 화면 속 그림들의 영어 이름부터 정확히 아는 것이 시작입니다. 이미 안다고 생각하는 단어 속에 숨겨진 <strong>'진짜 의미'</strong>를 찾아봅시다.</p>
<br />

<h2 id="1-내-책상-위의-영어-hardware">1. 내 책상 위의 영어 (Hardware)</h2>
<p>컴퓨터의 겉모습(하드웨어)과 관련된 용어입니다. 너무 쉽다고요? 개발자가 되면 이 단어들이 조금 다르게 쓰입니다.</p>
<h3 id="🔌-port-포트">🔌 Port (포트)</h3>
<ul>
<li><strong>사전적 의미:</strong> 항구</li>
<li><strong>전산적 의미:</strong> <strong>데이터가 드나드는 연결 통로(구멍)</strong></li>
<li><strong>설명:</strong> 배가 항구(Port)에 정박해서 물건을 내리고 싣죠? 컴퓨터에서도 마우스, 키보드, 인터넷 선이 연결되는 구멍을 모두 '포트'라고 부릅니다.<ul>
<li><em>USB Port:</em> USB를 꽂는 항구</li>
<li><em>Charging Port:</em> 충전기를 꽂는 항구</li>
<li><em>(심화)</em> 나중에 서버 개발을 배우면 <strong>&quot;8080 포트 열어놔&quot;</strong>라는 말을 듣게 됩니다. 물리적인 구멍뿐만 아니라, 프로그램끼리 통신하는 '가상의 문'도 포트라고 부릅니다.</li>
</ul>
</li>
</ul>
<h3 id="🖥️-monitor-vs-screen-vs-display">🖥️ Monitor vs Screen vs Display</h3>
<ul>
<li><strong>Monitor:</strong> TV처럼 생긴 기계 덩어리 전체를 말합니다.</li>
<li><strong>Screen:</strong> 실제로 빛이 나오는 '화면' 부분입니다.</li>
<li><strong>Display:</strong> 무언가를 '보여주는 장치'를 통틀어 말합니다.<ul>
<li>*&quot;모니터 전원 켜&quot; (기계)* vs *&quot;스크린에 띄워봐&quot; (화면)*</li>
</ul>
</li>
</ul>
<h3 id="🖱️-device-디바이스">🖱️ Device (디바이스)</h3>
<ul>
<li><strong>의미:</strong> 장치, 기기</li>
<li><strong>설명:</strong> 컴퓨터에 연결되는 모든 기계를 퉁쳐서 '디바이스'라고 합니다. 스마트폰, 태블릿, 프린터 모두 디바이스입니다.<ul>
<li><em>Input Device:</em> 입력 장치 (키보드, 마우스)</li>
<li><em>Output Device:</em> 출력 장치 (모니터, 스피커)</li>
</ul>
</li>
</ul>
<br />

<h2 id="2-화면-속의-기본-단어-basic-ui">2. 화면 속의 기본 단어 (Basic UI)</h2>
<p>컴퓨터를 켜면 윈도우나 맥 화면(UI: User Interface)이 보입니다. 여기에 있는 그림들에도 다 영어 이름이 있습니다.</p>
<h3 id="📂-directory-디렉토리-vs-folder-폴더">📂 Directory (디렉토리) vs Folder (폴더)</h3>
<ul>
<li><strong>Folder:</strong> 서류철. 윈도우 화면에서 보이는 노란색 아이콘을 말합니다.</li>
<li><strong>Directory:</strong> 주소록/목록. 개발자들은 폴더라는 말보다 **'디렉토리'**라는 말을 더 많이 씁니다.<ul>
<li>엄밀히 따지면 조금 다르지만, 초보 단계에서는 **&quot;폴더 = 디렉토리&quot;**라고 생각해도 무방합니다.</li>
<li>*&quot;그 파일 어느 디렉토리에 있어?&quot;* = *&quot;그 파일 어느 폴더에 들어있어?&quot;*</li>
</ul>
</li>
</ul>
<h3 id="🗑️-trash-트래시--bin-빈">🗑️ Trash (트래시) / Bin (빈)</h3>
<ul>
<li><strong>의미:</strong> 쓰레기통</li>
<li><strong>설명:</strong> 파일을 삭제하면 바로 사라지는 게 아니라 이곳으로 갑니다. 맥(Mac)에서는 <strong>Trash</strong>, 윈도우 영문판에서는 <strong>Recycle Bin</strong>(재활용 통)이라고 합니다.</li>
</ul>
<h3 id="💻-desktop-데스크탑">💻 Desktop (데스크탑)</h3>
<ul>
<li><strong>어원:</strong> Desk(책상) + Top(위)</li>
<li><strong>의미:</strong> 바탕화면</li>
<li><strong>설명:</strong> 우리가 책상 위에 공책과 필통을 꺼내두고 공부하죠? 컴퓨터 화면에서 아이콘들을 꺼내두는 기본 배경을 '책상 위' 같다고 해서 <strong>Desktop</strong>이라고 부릅니다.</li>
</ul>
<h3 id="🪟-window-윈도우">🪟 Window (윈도우)</h3>
<ul>
<li><strong>의미:</strong> 창문</li>
<li><strong>설명:</strong> 프로그램을 실행했을 때 네모나게 뜨는 틀을 말합니다. 우리는 이 '창문'을 통해 프로그램의 내용을 들여다봅니다.</li>
</ul>
<br />

<h2 id="3-위치와-방향-position">3. 위치와 방향 (Position)</h2>
<p>개발(특히 웹사이트 만들기)을 하다 보면 위치를 지정해야 할 일이 정말 많습니다.</p>
<ul>
<li><strong>Top / Bottom:</strong> 위 / 아래 (맨 위, 맨 아래)</li>
<li><strong>Left / Right:</strong> 왼쪽 / 오른쪽</li>
<li><strong>Center / Middle:</strong> 중앙 (가로 중앙, 세로 중앙)</li>
<li><strong>Drag &amp; Drop:</strong><ul>
<li><strong>Drag:</strong> (마우스를 누른 채) 질질 끌다</li>
<li><strong>Drop:</strong> (원하는 곳에) 툭 떨어뜨리다</li>
<li>파일을 옮길 때 하는 그 동작을 말합니다.</li>
</ul>
</li>
</ul>
<br />
