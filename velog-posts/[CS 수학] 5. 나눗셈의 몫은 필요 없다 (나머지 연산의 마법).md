<h1 id="5-나눗셈의-몫은-필요-없다-나머지-연산의-마법">5. 나눗셈의 몫은 필요 없다 (나머지 연산의 마법)</h1>
<p>초등학교 때 나눗셈을 배우면 대부분 <strong>‘몫’</strong>에 관심을 가집니다.
“사과 10개를 3명이 나눠 가지면?” → “3개씩 먹을 수 있다.”
하지만 컴퓨터 과학에서는 반대입니다.</p>
<p>컴퓨터가 관심을 가지는 것은 <strong>“그래서 얼마가 남지?”</strong>입니다.</p>
<p>바로 이 <em>나머지</em>가 보안, 자료구조, 알고리즘, 시스템 설계 등 곳곳에서 핵심 역할을 하기 때문입니다.</p>
<hr />
<h1 id="0-일상-속의-나머지-연산">0. 일상 속의 나머지 연산</h1>
<blockquote>
<p><strong>“13시는 1시입니다.”</strong></p>
</blockquote>
<p>이 문장이 자연스러운 이유는 우리가 이미 일상에서 <strong>나머지(Modulo)</strong> 개념을 사용하기 때문입니다.
시계는 12 이후 다시 1로 돌아오고, 요일은 7마다 반복됩니다.</p>
<p>즉, <strong>숫자가 무한히 커지는 대신 일정한 범위 안에서 순환하도록 만드는 것</strong>
이것이 <code>%</code> 연산의 본질입니다.</p>
<hr />
<h1 id="1-짝수홀수-판별--가장-단순한-패턴-인식">1. 짝수/홀수 판별 – 가장 단순한 패턴 인식</h1>
<p>짝수인지 홀수인지 어떻게 판단할까요?</p>
<p>“2로 나눴을 때 나머지가 0이면 짝수”라는 간단한 규칙을 사용합니다.</p>
<pre><code class="language-python">num = 15
if num % 2 == 0:
    print(&quot;짝수&quot;)
else:
    print(&quot;홀수&quot;)</code></pre>
<p>이 개념을 확장하면 다음과 같은 로직도 쉽게 만들 수 있습니다.</p>
<ul>
<li>3번마다 배너 띄우기 → <code>count % 3 == 0</code></li>
<li>100번째 방문자 이벤트 → <code>visit % 100 == 0</code></li>
<li>반복되는 패턴 처리</li>
</ul>
<p>나머지는 <strong>주기(Period)</strong>를 판별하는 가장 직관적이고 빠른 도구입니다.</p>
<hr />
<h1 id="2-hash-table--무한한-데이터를-유한한-공간에-담기">2. Hash Table – 무한한 데이터를 유한한 공간에 담기</h1>
<p>면접 단골 질문인 해시 테이블의 핵심도 바로 <strong>나머지 연산</strong>입니다.</p>
<p>예를 들어 책장 칸이 100개(0~99)뿐인데, 책 번호는 59283, 12034처럼 매우 큽니다.
이때 책을 어디에 꽂아야 할까요?</p>
<p>방법은 간단합니다.</p>
<ul>
<li>책 번호 59283</li>
<li>칸 수 100</li>
<li>위치 = <code>59283 % 100 = 83</code></li>
</ul>
<p>즉,</p>
<p><strong>아무리 큰 수라도 작은 범위로 변환하는 기술</strong>
→ 이것이 해싱(Hashing)의 토대입니다.</p>
<p>이 방식 덕분에:</p>
<ul>
<li>검색 속도는 매우 빠르고</li>
<li>공간은 제한적이어도</li>
<li>데이터는 무제한처럼 저장할 수 있습니다.</li>
</ul>
<hr />
<h1 id="3-암호학--보안의-근본-원리">3. 암호학 – 보안의 근본 원리</h1>
<p>인터넷 뱅킹이나 메신저에서 사용하는 RSA 암호 등 현대 암호학 대부분은 <strong>엄청나게 큰 소수에 대한 나머지 연산</strong>을 사용합니다.</p>
<p>이유는 단순하지만 강력합니다.</p>
<h3 id="정방향-계산은-쉽지만">정방향 계산은 쉽지만</h3>
<h3 id="역방향-추적은-거의-불가능하기-때문입니다">역방향 추적은 거의 불가능하기 때문입니다.</h3>
<p>예를 들어:</p>
<ul>
<li>( 5^3 % 7 ) 은 쉽게 계산됩니다.</li>
<li>하지만 ( 5^x % 7 = 6 ) 이 되게 하는 x를 찾는 것은 매우 어렵습니다.</li>
</ul>
<p>이런 성질을 <strong>일방향성(one-way)</strong>이라 부르며,
오늘날의 암호 시스템은 모두 이 성질을 바탕으로 동작합니다.</p>
<hr />
<h1 id="4-원형-큐circular-queue--배열을-무한처럼-쓰는-기술">4. 원형 큐(Circular Queue) – 배열을 무한처럼 쓰는 기술</h1>
<p>일반적인 큐는 데이터가 들어오다 보면 배열 끝에 도달합니다.
그러면 더 이상 데이터를 넣을 수 없겠죠?</p>
<p>하지만 나머지 연산을 사용하면 배열의 끝과 처음이 연결됩니다.</p>
<pre><code>next_index = (current_index + 1) % 배열_크기</code></pre><p>이렇게 하면 아래와 같은 장점을 얻습니다.</p>
<ul>
<li>메모리를 계속 재사용할 수 있고</li>
<li>추가적인 확장 없이</li>
<li>무한히 데이터를 처리하는 것처럼 만들 수 있습니다.</li>
</ul>
<p>원형 큐는 네트워크 패킷 처리, 키보드 입력 버퍼 등 <strong>실시간 처리 시스템</strong>에서 매우 중요한 구조입니다.</p>

<h1 id="오늘의-요약">오늘의 요약</h1>
<ol>
<li>컴퓨터는 몫보다 <strong>나머지를 더 중요하게 사용</strong>한다.</li>
<li><code>%</code> 연산자는 숫자를 <strong>특정 범위 안에서 순환시키는 도구</strong>이다.</li>
<li>해시 테이블은 나머지 연산을 이용해 큰 숫자를 <strong>작은 인덱스로 사상(mapping)</strong>한다.</li>
<li>암호학은 나머지 연산의 <strong>역추적하기 어려운 성질</strong>을 기반으로 한다.</li>
<li>원형 큐는 <code>%</code> 연산을 이용해 배열을 <strong>순환 구조</strong>로 만든다.</li>
</ol>
