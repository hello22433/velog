<h3 id="컴퓨팅-시스템">컴퓨팅 시스템</h3>
<p>여러 개의 컴포넌트들이 하나의 목적을 가지고 모여 이루는 체계.</p>
<h3 id="하드웨어">하드웨어</h3>
<p>컴퓨팅 시스템을 구성하는 물리적인 요소들</p>
<h3 id="소프트웨어">소프트웨어</h3>
<p>지시하는 명령어들로 구성된 프로그램들</p>
<h3 id="컴퓨터-시스템의-계층들">컴퓨터 시스템의 계층들</h3>
<p>Information 보통 이진 정보
Hardware 정보를 이해할 수 있는 하드웨어
Programming 하드웨어를 제어하는 프로그래밍
Operating systems (자원의 효율적 배분의 주체 및 환경, 하나의 두뇌로 여러가지 일하게 만듬)
Applications (하드웨어를 직접 다루지 아니하는 소프트웨어 계층, 여러 가지 도메인)
Communications 통신 계층</p>
<h3 id="추상화">추상화</h3>
<p>복잡한 세부사항이 제거된 개념 모델 (핸들을 돌리는 과정 아래에는 여러가지와 연결 되어서 좌우로 가는 결과를 만든다. 핸들을 돌려 방향을 바꾸는 것이 추상화한 개념, 이를 쓸 때에 자동차 내부 구조를 몰라도 됨)</p>
<h3 id="컴퓨팅의-초기-역사">컴퓨팅의 초기 역사</h3>
<p>주판 : 수치를 기록하는 초기 장비
Blaise Pascal : 사칙연산을 할 수 있는 기계식 계산기
Joseph Jacquard : 자커드의 방칙기, 구멍이 뚫려 있는 카드
Charlse Babbage : 해석기관, 메모리 개념 등장
Ada Lovelace : 최초의 프로그래머 (사실 컴퓨터가 없었으니 알고리즘 연구가) , 루프 개념 등장
AlanTuring : 튜링 기계, 인공지능 시험, 현대 컴퓨터의 연산 방식을 최초로 완성한 사람
Harvard Make I, ENIAC, UNIVAC I : 수학,물리학,경제학에 큰 영향을 준 컴퓨터들</p>
<h3 id="제1세대-하드웨어-1951-1959">제1세대 하드웨어 1951-1959</h3>
<p>진공관 : 부피가 크고, 많은 열을 발생시켜 믿을 수 없었음
마그네틱 드럼 : 입출력 헤드 밑에서 돌아가는 기억 장치
카드리더 -&gt; 마그네틱 테이프 장치 : 순차적 보조 기억 장치</p>
<h3 id="제2세대-하드웨어-1959-1965">제2세대 하드웨어 1959-1965</h3>
<p>트랜지스터 : 진공관 대체, 빠르고 작으며 오래가고 가격도 저렴함
마그네틱 코어 : 마그네틱 드럼 대체, 정보 처리가 즉시 이루어짐
마그네틱 디스크 : 마그네틱 테이프 대체, 데이터에 직접 접근 가능함</p>
<h3 id="제3세대-하드웨어-1965-1971">제3세대 하드웨어 1965 1971</h3>
<p>입적회로 : 회로기판을 대체, 더 작고 ,싸며 , 빠르고 , 더 믿을만함
트렌지스터 : 현재는 기억장치구성에 쓰임
터미널 : 키보드와 스크린으로 구성된 입출력장치</p>
<h3 id="제4세대-하드웨어-1971-">제4세대 하드웨어 1971-?</h3>
<p>대규모 집적회로 : 칩 기술의 커다란 발전
개인용 컴퓨터, 컴퓨터의 상용화, 워크스테이션 : 개인용 컴퓨터와 워크스테이션이 나타남, 새로운 기업들이 등장 : 애플,썬,델...
노트북, 태블릿 컴퓨터, 그리고 스마트폰 : 개인이 컴퓨터를 휴대하게 됨</p>
<h3 id="병렬-컴퓨팅과-네트워킹">병렬 컴퓨팅과 네트워킹</h3>
<p>병렬 컴퓨팅 : 컴퓨터들은 서로 연결되어 처리 속도를 높여주는 중앙 처리장치와 (또는) 기억 장치들에 의존한다.
네트워킹 : Ethernet이 소형 컴퓨터들을 연결하여 자원 공유를 가능하게 함, 1980년대 후반에 파일 서버들이 PC들을 연결해 줌
ARPANET과 랜 -&gt; 인터넷</p>
<h3 id="제1세대-소프트웨어-1951-1959">제1세대 소프트웨어 1951 1959</h3>
<p>기계언어 : 컴퓨터 프로그램들이 이진수로 작성됨
어셈블리어와 번역기 : 연상 기호로 프로그램을 작성하여 기계어로 번역함
프로그래머들의 변화 : 두 그룹으로 나뉘게 됨: 응용 프로그래머와 시스템 프로그래머</p>
<h3 id="어셈블리-기계">어셈블리 기계</h3>
<p>시스템 프로그래머들은 어셈블러(번역기)를 작성한다.
응용 프로그래머들은 문제해결을 위해 어셈블리 언어를 사용한다.</p>
<h3 id="제-2세대-소프트웨어-1959-1965">제 2세대 소프트웨어 1959 1965</h3>
<p>고급언어 : 영어와 닮은 문장들의 사용으로 프로그래밍이 쉬워졌다. Fortran, COBOL, Lisp
시스템프로그래머들은 고급 언어용 번역기를 작성한다.
응용프로그래머들은 문제 해결을 위해 고급언어를 사용한다.</p>
<h3 id="제3세대-소프트웨어-1965-1971">제3세대 소프트웨어 1965 1971</h3>
<p>시스템소프트웨어 : 유틸리티 프로그램, 언어번역기(컴파일러), 운영체제가 시행할 프로그램과 시간을 결정
사용자와 하드웨어의 분리 : 컴퓨터 프로그래머들이 (프로그래머가 아닌) 일반인들이 사용할 수 있는 프로그램 작성</p>
<h3 id="제4세대-소프트웨어">제4세대 소프트웨어</h3>
<p>구조화된 프로그래밍 : Pascal C++
사용자를 위한 새로운 응용 소프트웨어 : 스프레드시트, 문서편집기, 데이터베이스 관리 시스템</p>
<h3 id="제5세대-소프트웨어">제5세대 소프트웨어</h3>
<p>마이크로소프트 : 윈도우즈 운영체제와 마이크로소프트 사의 다른 응용 프로그램들이 시장을 점령함
객체지향 설계 : 객체 데이터의 서열을 따르는 방식(Java)
월드와이드웹 : 인터넷을 통한 전세계와의 소통을 쉽게 만들어줌
새로운 사용자들 : 새로운 사용자는 컴퓨터에 대한 지식을 필요로 하게 됨</p>
<h3 id="도구로서의-컴퓨팅">도구로서의 컴퓨팅</h3>
<p>응용프로그래머들은 도구(시스템프로그램)을 이용해서 일반인을 위한 프로그램을 만든다.</p>

꼭 필요한 네 가지 기술 : 알고리즘적인 사고, 표현, 프로그래밍, 설계</p>
